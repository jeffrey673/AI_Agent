## 슬라이드 2: 왜 만들었는가?
**기존 업무의 문제점**
- 매출 데이터 확인: BigQuery 콘솔 접속 → SQL 작성 → 결과 해석 (20분+)
- 사내 문서 검색: Notion 페이지 뒤지기, 어디 있는지 모름
- CS 응대: 스프레드시트 검색, 제품별 탭 찾기
- Google Workspace: 메일/일정 확인하려면 앱 전환

**AI Agent 도입 후**
- "이번 달 인도네시아 매출 알려줘" → 10초 내 답변 + 차트
- 자연어로 모든 업무 데이터에 즉시 접근

## 슬라이드 3: 시스템 개요 (아키텍처 다이어그램)
**전체 아키텍처 그림을 그려주세요:**

```
사용자 (웹 브라우저)
    ↓ [HTTPS]
FastAPI Server (Python)
    ↓
┌─────────────────────────────┐
│    LangGraph Orchestrator   │
│    (질문 의도 자동 분류)      │
└──┬──┬──┬──┬──┬──┬───────────┘
   │  │  │  │  │  │
   ▼  ▼  ▼  ▼  ▼  ▼
  BQ RAG CS GWS Multi Direct
  │   │   │   │    │    │
  ▼   ▼   ▼   ▼    ▼    ▼
BigQuery Notion Sheets Gmail Claude
(매출DB) (문서) (CS) (메일) (LLM)
```

- 사용자 질문 → AI가 자동으로 6개 경로 중 최적 라우팅
- SQL 자동 생성 → BigQuery 실행 → 답변 + 차트 자동 생성

## 슬라이드 4: 6가지 라우팅 경로
**질문 유형별 자동 분류 (예시 포함)**

| 라우트 | 질문 예시 | 처리 방식 |
|--------|----------|----------|
| 📊 BigQuery | "이번 달 매출", "국가별 TOP 5" | 자연어→SQL 자동 변환→실행→차트 |
| 📋 Notion | "틱톡샵 접속 방법", "출장 가이드" | 사내 문서 벡터 검색 |
| 🧴 CS Q&A | "센텔라 앰플 성분", "사용법" | 739개 CS 데이터 키워드 검색 |
| 📧 GWS | "오늘 일정", "메일 확인" | Gmail/Calendar/Drive API |
| 📈 Multi | "매출 하락 원인 분석" | BQ 데이터 + 웹검색 종합 |
| 💬 Direct | "안녕", "날씨", "회사 소개" | Claude Sonnet 실시간 스트리밍 |

## 슬라이드 6: BigQuery 데이터 — 13개 테이블

- 🔍 아마존 검색 — CTR/전환율/검색쿼리
- 📱 메타 광고 — Facebook/Instagram 광고 라이브러리
- 🛒 플랫폼 순위 — 경쟁사 포함 가격/순위

## 슬라이드 10: 실시간 스트리밍 (ChatGPT급)
**스트리밍 비교 다이어그램:**

| | 이전 | 현재 |
|---|---|---|
| 첫 텍스트 | 9.5초 후 한꺼번에 | **2초부터 타이핑** |
| 렌더링 | 전체 완성 후 | **50ms 간격 실시간** |
| 체감 | 빈 화면 10초 | **ChatGPT급 타이핑** |

기술:
- Claude Sonnet 실시간 스트리밍 (TTFB 2초)
- Thread → asyncio.Queue → SSE 실시간 전송
- 블링킹 커서 애니메이션

## 슬라이드 11: 보안 & 권한 관리
**보안 아키텍처 다이어그램:**

- AD 연동 로그인 (328명 전직원)
- JWT httpOnly 쿠키 인증
- 그룹별 데이터 접근 제한 (brand_filter)
  - SK 그룹: SK/CL/CBT 브랜드만 조회
  - DD 그룹: DD 브랜드만 조회
  - Admin: 전체 접근
- SQL 안전장치: SELECT ONLY, 테이블 화이트리스트
- GWS: 개인별 OAuth (타인 메일 접근 불가)
