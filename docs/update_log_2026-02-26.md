### 1. BQ-39: Team_NEW 컬럼 값 학습 + 빈 결과 안내 개선

#### 1-A. 문제
- "GM_WEST팀 국가별 매출 분포" 쿼리 → SQL에서 `Team_NEW = 'GM_WEST'` 조건 생성
- 실제 Team_NEW 컬럼에 'GM_WEST'는 존재하지 않음 → 결과 0행
- LLM이 "데이터가 없습니다"만 반환, 올바른 팀명 안내 없음

#### 1-B. 수정
1. **SQL 프롬프트 Team_NEW 값 학습** (`prompts/sql_generator.txt`):
   - BigQuery에서 실제 Team_NEW 고유값 14개 조회하여 프롬프트에 반영
   - GM_EAST1, GM_EAST2, GM_Ecomm, GM_MKT, CBT, JBT, KBT, BCM, B2B1, B2B2, DD_DT1, DD_DT2, OP, 기타
   - "GM_WEST는 존재하지 않음" 경고 명시
2. **빈 결과 value hints** (`app/agents/sql_agent.py`):
   - SQL에 Team_NEW/Country 컬럼 참조 시 유효값 목록을 Flash 프롬프트에 전달
   - Flash가 "GM_WEST는 존재하지 않습니다. GM_EAST1, GM_EAST2를 사용하세요" 안내 생성

#### 1-C. 수정 파일
| 파일 | 변경 |
|------|------|
| `prompts/sql_generator.txt` | Team_NEW 14개 유효값 + 한국어 설명 추가 |
| `app/agents/sql_agent.py` | `_value_hints` 빈 결과 핸들러에 컬럼 유효값 전달 |

### 2. NT-14: Notion 검색 정확도 개선 (4건)

#### 2-A. 문제
- "반품 접수부터 완료까지 전체 플로우" → "틱톡샵 접속 방법" 반환
- 원인 1: 한국어 조사 "접수부터", "완료까지"가 키워드 매칭 실패
- 원인 2: LLM 폴백이 관련 없는 페이지도 강제 선택
- 원인 3: 답변 프롬프트가 무관한 내용으로 답변 허용

#### 2-B. 수정 (`app/agents/notion_agent.py`)
1. **한국어 조사 제거**: "접수부터"→"접수", "완료까지"→"완료" (18개 조사 패턴)

3. **미발견 메시지 개선**: 접근 가능한 문서 목록 표시 + 재질문 안내
4. **답변 프롬프트 강화**: 규칙 7, 9에서 질문 주제와 무관한 문서 내용으로 답변 금지

#### 2-C. 수정 파일
