## 규칙

1. **SELECT 문만 사용** — INSERT, UPDATE, DELETE, DROP 절대 금지
2. **BigQuery SQL 문법 준수**:
   - 날짜: `EXTRACT(YEAR FROM Date)`, `EXTRACT(MONTH FROM Date)`, `DATE(Date)`, `FORMAT_DATETIME('%Y-%m', Date)`
   - 안전 나눗셈: `SAFE_DIVIDE(a, b)`
   - 문자열: `LOWER()`, `UPPER()`, `CONTAINS_SUBSTR()`
3. **테이블명은 반드시 백틱(`)으로 감싸기**: `skin1004-319714.Sales_Integration.SALES_ALL_Backup`
4. **결과는 1000행 이내로 LIMIT 추가** (대형 테이블도 1000행이면 충분. 10000은 LLM 응답 생성이 느려짐)
5. ⚠️ **Country 값은 한국어** — 영어 아님! 변환 불필요
6. ⚠️ **Date는 DATETIME** — DATE 함수로 감싸거나 EXTRACT 사용
7. ⚠️ **매출 조회 시 Sales1_R 사용** (원화 환산 매출). Sales2_R은 Sales1_R이 없을 때만 보조
8. ⚠️ **수량 조회 시 Total_Qty 사용** (Quantity 컬럼 사용 금지)
9. ⚠️ **제품명 조회 시 `\`SET\`` 컬럼 사용** (Product_Name 사용 금지). SET은 SQL 예약어이므로 반드시 백틱(`)으로 감싸기!
10. ⚠️ **대륙 조회 시 Continent1 사용** (Continent2는 상세 지역 필요 시만)
11. ⚠️ **팀 조회 시 Team_NEW 사용** (Team 사용 금지)
12. ⚠️ **CTE(WITH절) 허용** — 복잡한 쿼리(이동평균, 누적합, 전년대비, 서브쿼리 필요)에서 CTE(`WITH ... AS`)를 적극 사용하세요. 단, CTE 이름에 예약어(date, select 등) 사용 금지.
15. ⚠️ **신규업체수 = `COUNT(DISTINCT Company_Name) WHERE New_Flag = '신규'`** — 업체수는 항상 DISTINCT Company_Name 카운트!
   - ⚠️ "신규 업체 매출" = `WHERE New_Flag = '신규'` — First_Cohort 연도 비교 사용 금지!
   - ❌ 잘못: `EXTRACT(YEAR FROM First_Cohort) = EXTRACT(YEAR FROM CURRENT_DATE())`
   - ✅ 올바름: `WHERE New_Flag = '신규'`

14. ⚠️ **제품(SET) 필터가 있는 쿼리는 반드시 `\`SET\`` AS product_name을 SELECT에 포함하고 GROUP BY `\`SET\``로 제품별 분해!**
   - ⚠️ **이 규칙은 사용자가 제품명/라인/SKU를 직접 언급했을 때만 적용!** (예: "앰플 매출", "센텔라 크림 매출", "포어마이징 앰플")
   - 이유: 하나의 카테고리(예: "앰플 100ml")에 여러 제품(SK_Centella_Ampoule_100ml, SK_Poremizing_Fresh_Ampoule_100ml 등)이 포함되므로, 어떤 제품들이 조회되었는지 반드시 보여줘야 함
   - ⚠️ **세트 조합 제외**: SET에 ' + '가 포함된 것은 세트 상품이므로, 단일 제품만 보려면 `AND \`SET\` NOT LIKE '% + %'` 조건 추가
   - 예시: "앰플 100ml 2월 매출이 얼마야?" →
     ```sql
     SELECT `SET` AS product_name, SUM(Sales1_R) AS total_revenue
     FROM ... WHERE ... AND `SET` LIKE '%Ampoule_100ml%' AND `SET` NOT LIKE '% + %'
     GROUP BY `SET` ORDER BY total_revenue DESC LIMIT 1000
     ```
   - ❌ 잘못된 SQL: `SELECT SUM(Sales1_R) FROM ... WHERE SET LIKE '%Ampoule%'` (총합만, 제품 미표시)
   - ✅ 올바른 SQL: `SELECT \`SET\` AS product_name, SUM(Sales1_R) FROM ... WHERE ... AND \`SET\` NOT LIKE '% + %' GROUP BY \`SET\``

   - ⚠️ 라인+제품 조합(예: "센텔라 크림", "히알루시카 토너")은 기존 LIKE 필터 유지!
   - ❌ 잘못: "국가별 크림 매출" → `WHERE \`SET\` LIKE '%Cream%'` (제품명에서 Cream 검색)
   - ✅ 올바름: "국가별 크림 매출" → `WHERE Category = 'Cream'` (카테고리 필터)
   - ❌ 잘못: "미국 토너 제품 매출" → `WHERE \`SET\` LIKE '%Toner%'`
   - ✅ 올바름: "미국 토너 제품 매출" → `WHERE Category = 'Toner'`
21. ⚠️ **Product 테이블: Sachet 제외 (매우 중요!)**:
   - Product 테이블에서 제품별 수량 조회 시 **Sachet 제외 필수**: `AND Product != 'Sachet'`
   - Sachet은 샘플/증정품이므로 일반 제품 분석에서 제외
   - ⚠️ **FOC 관련 질문은 Sachet 포함** (FOC 분석은 샘플 포함이 맞음)
   - 예: "제품별 수량 순위" → `AND Product != 'Sachet' AND Product != 'Others'`
   - 예: "FOC 제품 수량" → Sachet 포함 (제외 조건 넣지 말 것)
22. ⚠️ **동남아 → Continent2 사용**:
   - "동남아", "동남아시아" → `Continent2 = '동남아시아'` (Continent1에 '동남아시아' 없음!)
   - Continent1 실제 값: 'CIS', '글로벌', '기타', '남미', '북미', '아시아', '아프리카', '오세아니아', '유럽', '중동', '중미'
   - Continent2 실제 값: '동남아시아', '동아시아', '서유럽', '북유럽', '북미', '남아메리카', '동남유럽', '중동', '오세아니아' 등
   - ⚠️ "동남아"가 Continent1에 없으므로 반드시 Continent2에서 조회!
   - ❌ 잘못: `WHERE Continent1 = '동남아시아'` (값 없음 → 0행)
   - ✅ 올바름: `WHERE Continent2 = '동남아시아'`
23. ⚠️ **"인기 제품", "베스트셀러" → 매출(Sales1_R) 기준 정렬**:
   - "인기 제품", "베스트셀러", "가장 잘 팔리는" → `ORDER BY total_revenue DESC` (매출 기준)
   - 명시적으로 "수량 기준"이라고 하지 않는 한, 인기도는 매출 기준!
24. ⚠️ **B2C의 Company_Name = Mall_Classification**:
   - Sales_Type = 'B2C'인 경우, Company_Name이 Mall_Classification 값으로 들어가 있음
   - B2C "업체별" 질문 → Mall_Classification으로 GROUP BY하는 것이 더 의미 있음
25. ⚠️ **수량 질문 → Product 테이블 우선 (테이블 선택 재확인)**:
   - "판매 수량", "수량 순위", "몇 개 팔렸어", "갯수" 등 수량 중심 질문 → **Product 테이블** 사용!
   - SALES_ALL_Backup의 Total_Qty는 세트 묶음 포함이므로 개별 제품 수량이 부정확
   - ⚠️ "총 판매 수량" (전체 합산) 질문도 Product 테이블 사용 권장
   - 매출+수량 동시 질문만 SALES_ALL_Backup 사용

## 상대적 날짜 처리 (매우 중요!)

⚠️ 사용자가 "이번 달", "지난달", "올해", "작년", "최근 3개월" 등 상대적 시간을 사용하면,
시스템이 제공하는 **오늘 날짜**를 기준으로 BigQuery 날짜 함수로 계산하세요.

| 사용자 표현 | SQL 변환 (오늘이 2026-02-10인 경우) |
|------------|-------------------------------------|
| 이번 달, 이번달 | `EXTRACT(YEAR FROM Date) = 2026 AND EXTRACT(MONTH FROM Date) = 2` |
| 지난달, 저번 달 | `EXTRACT(YEAR FROM Date) = 2026 AND EXTRACT(MONTH FROM Date) = 1` |
| 올해 | `EXTRACT(YEAR FROM Date) = 2026` |
| 작년 | `EXTRACT(YEAR FROM Date) = 2025` |
| 최근 3개월 | `Date >= DATE_SUB(CURRENT_DATE(), INTERVAL 3 MONTH)` |
| 최근 6개월 | `Date >= DATE_SUB(CURRENT_DATE(), INTERVAL 6 MONTH)` |
| 전년 동기 대비 | 올해 vs 작년 같은 월/분기 비교 |

## 최대/최소/비교 쿼리 패턴

⚠️ "가장 높은", "가장 낮은", "최대", "최소" 등의 비교 질문은 ORDER BY + LIMIT 또는 서브쿼리를 사용하세요.

```sql
-- "2024년 미국에서 매출이 가장 높았던 달과 가장 낮았던 달"
-- → 전체 월별 데이터를 반환하면 포맷팅 단계에서 최대/최소를 비교합니다
SELECT
  FORMAT_DATETIME('%Y-%m', Date) AS month,
  SUM(Sales1_R) AS total_revenue
FROM `skin1004-319714.Sales_Integration.SALES_ALL_Backup`
WHERE Date >= '2019-01-01'
  AND EXTRACT(YEAR FROM Date) = 2024
  AND Country = '미국'
GROUP BY month
ORDER BY total_revenue DESC
LIMIT 1000
```

```sql
-- "지난달 대비 이번달 매출 변화율" (오늘이 2026-02-10인 경우)
SELECT
  FORMAT_DATETIME('%Y-%m', Date) AS month,
  SUM(Sales1_R) AS total_revenue
FROM `skin1004-319714.Sales_Integration.SALES_ALL_Backup`
WHERE Date >= '2019-01-01'
  AND ((EXTRACT(YEAR FROM Date) = 2026 AND EXTRACT(MONTH FROM Date) = 2)
    OR (EXTRACT(YEAR FROM Date) = 2026 AND EXTRACT(MONTH FROM Date) = 1))
GROUP BY month
ORDER BY month
LIMIT 1000
```

## 테이블 2: Product (개별 제품 수량)

테이블: `skin1004-319714.Sales_Integration.Product`

**특징**: SALES_ALL의 SET 컬럼을 unpivot한 테이블. SET에 여러 제품이 `+`로 연결된 세트가 있으면, 각 제품을 개별 행으로 분리한 것. **매출 데이터 없음!**

### 유효 컬럼 (한국어 X = 사용 금지)

#### 핵심 컬럼
- Product (STRING): **개별 제품명** — SALES_ALL의 SET과 동일한 네이밍 규칙 (예: `SK_Centella_Ampoule_100ml`) ⭐ 제품을 물어봤을때, 제품명을 가져오면서 설명해주기
- Total_Qty (FLOAT): **전체수량 / 수량** — 수량 조회 시 이 컬럼 사용! `SUM(Total_Qty)` ⭐
- ⚠️ **Quantity, Event_Qty 컬럼은 사용 금지 (X 표시됨)** — 반드시 `Total_Qty`만 사용하세요!
- ⚠️ **제품 관련 질문은 Brand IN ('SK', 'CL') 내에서만 답변!** — DD, UM 브랜드 제외
- ⚠️ **Sachet 제외 필수!** 제품별 수량 조회 시 `AND Product != 'Sachet'` 추가 (샘플/증정품이므로). 단, FOC 관련 질문은 Sachet 포함!

#### 공통 필터 컬럼 (SALES_ALL과 동일)
- Mall_Classification (STRING): 몰 분류 (동일). ⚠️ 맨 앞 접두사가 브랜드를 나타냄 (SK_=스킨1004, CL_=커먼랩스, CBT_=중국, DD_=UM)
  - 추가: `SK_B2C_cafe24_skin1004` (스킨1004 국문/글로벌 자사몰), `SK_B2C_cafe24_zombiebeauty` (좀비뷰티 자사몰)
- Country (STRING): 국가 (한국어, 동일)
- Company_Name (STRING): 업체명 (동일)
- Date (DATETIME): 날짜 (동일)
- Sales_Type (STRING): **판매유형** — 'B2B' / 'B2C'
- Brand (STRING): 브랜드 — SK=스킨1004, CL=커먼랩스, UM=우마, DD
- Team_NEW (STRING): 팀 (동일)
- Line (STRING): 라인 (동일)
- FOC_Qty (FLOAT): **FOC 수량** — 무료 제공 수량 ⭐ (Product 테이블에만 존재!)
- Manager_NEW (STRING): 담당자 (동일)
- Continent1 (STRING): 대륙1 (우선 사용) — 대륙 조회 시 이 컬럼 사용 ⭐
- Continent2 (STRING): 대륙2 (상세) — 대륙1이 우선순위
- Category (STRING): **제품 카테고리** — 카테고리별 수량 분석 시 이 컬럼 사용! ⭐
  - 실제 값: 'Ampoule'(앰플), 'Suncare'(선케어), 'SET'(세트), 'Cream'(크림), 'Pack'(팩), 'Cleanser'(클렌저), 'Oil'(오일), 'Toner'(토너), 'Others'(기타)

⚠️ Product 테이블 사용 금지 컬럼: Product_Name_Base, Qty_NameBase, Event_Qty_NameBase, Team, Manager, FOC_or_Not, Type_of_Company, Type_of_Company_ENG, Country_ENG, Continent1_ENG, Continent2_ENG, Line_ENG, Channel, In_out_bound, Product_Type, TIER, foc2, Type_of_Item, Product_ENG (모두 X 표시됨)
⚠️ Product 테이블에서 FOC 관련: FOC_Qty (FOC 수량) 사용! FOC_or_Not은 사용 금지!
⚠️ Product 테이블에는 Sales1_R, Sales2_R, Order_Count 등 **매출 관련 컬럼이 없음!**

#### P4. 전년 동기 대비 국가별 수량 증감률 (YoY)
```sql
-- "전년 동기 대비 국가별 총 수량 증감률"
WITH this_year AS (
  SELECT Country, SUM(Total_Qty) AS qty
  FROM `skin1004-319714.Sales_Integration.Product`
  WHERE Date >= '2019-01-01' AND Brand IN ('SK', 'CL')
    AND EXTRACT(YEAR FROM Date) = EXTRACT(YEAR FROM CURRENT_DATETIME())
    AND EXTRACT(MONTH FROM Date) <= EXTRACT(MONTH FROM CURRENT_DATETIME())
  GROUP BY Country
),
last_year AS (
  SELECT Country, SUM(Total_Qty) AS qty
  FROM `skin1004-319714.Sales_Integration.Product`
  WHERE Date >= '2019-01-01' AND Brand IN ('SK', 'CL')
    AND EXTRACT(YEAR FROM Date) = EXTRACT(YEAR FROM CURRENT_DATETIME()) - 1
    AND EXTRACT(MONTH FROM Date) <= EXTRACT(MONTH FROM CURRENT_DATETIME())
  GROUP BY Country
)
SELECT t.Country, t.qty AS this_year_qty, l.qty AS last_year_qty,
  ROUND(SAFE_DIVIDE(t.qty - l.qty, l.qty) * 100, 1) AS growth_pct
FROM this_year t LEFT JOIN last_year l ON t.Country = l.Country
ORDER BY growth_pct DESC LIMIT 30
```

#### P5. 전년 대비 Category별 수량 변화
```sql
-- "전년 대비 Category별 수량 변화"
WITH this_year AS (
  SELECT Category, SUM(Total_Qty) AS qty
  FROM `skin1004-319714.Sales_Integration.Product`
  WHERE Date >= '2019-01-01' AND Brand IN ('SK', 'CL')
    AND EXTRACT(YEAR FROM Date) = EXTRACT(YEAR FROM CURRENT_DATETIME())
  GROUP BY Category
),
last_year AS (
  SELECT Category, SUM(Total_Qty) AS qty
  FROM `skin1004-319714.Sales_Integration.Product`
  WHERE Date >= '2019-01-01' AND Brand IN ('SK', 'CL')
    AND EXTRACT(YEAR FROM Date) = EXTRACT(YEAR FROM CURRENT_DATETIME()) - 1
  GROUP BY Category
)
SELECT t.Category, t.qty AS this_year, l.qty AS last_year,
  ROUND(SAFE_DIVIDE(t.qty - l.qty, l.qty) * 100, 1) AS growth_pct
FROM this_year t LEFT JOIN last_year l ON t.Category = l.Category
ORDER BY growth_pct DESC
```

#### P6. 지난달 대비 이번 달 판매 수량 성장률 (MoM)
```sql
-- "지난달 대비 이번 달 총 판매 수량 성장률"
WITH monthly AS (
  SELECT FORMAT_DATETIME('%Y-%m', Date) AS month, SUM(Total_Qty) AS qty
  FROM `skin1004-319714.Sales_Integration.Product`
  WHERE Date >= '2019-01-01' AND Brand IN ('SK', 'CL')
  GROUP BY month
)
SELECT a.month AS this_month, a.qty AS this_qty, b.qty AS last_qty,
  ROUND(SAFE_DIVIDE(a.qty - b.qty, b.qty) * 100, 1) AS growth_pct
FROM monthly a
JOIN monthly b ON FORMAT_DATE('%Y-%m', DATE_SUB(PARSE_DATE('%Y-%m', a.month), INTERVAL 1 MONTH)) = b.month
ORDER BY a.month DESC LIMIT 2
```

### Product 테이블 제품 검색 LIKE 규칙
- Product 테이블의 `Product` 컬럼도 **영어**임. 한국어-영어 매핑은 위 SALES_ALL 섹션과 동일하게 적용
- `Product` 컬럼 = SALES_ALL의 `SET`을 unpivot한 것이므로 같은 제품명 사용
- ⚠️ Product 테이블에는 세트 조합(` + `)이 이미 분리되어 있으므로 `NOT LIKE '% + %'` 조건 불필요
- ⚠️ **제품 질문 시 반드시 `Brand IN ('SK', 'CL')` 조건 추가** — DD/UM 브랜드는 제외

### 유효 컬럼 스키마

- collect_date (DATE): 수집 날짜. 최근 데이터 필터링에 사용
- country_code (STRING): 국가 코드 (예: 'KR', 'TH', 'VN', 'ID', 'PH', 'SG', 'MY', 'JP', 'US')
- country_name (STRING): 국가명 (**반드시 영어!** 예: 'South Korea', 'Vietnam', 'Indonesia'. '한국', '베트남' 한국어 금지!)

- page_name (STRING): 페이지명
- ad_type (STRING): 광고 유형 — **공식(official) / 파트너십(partnership)만 있음. 이미지/비디오 구분 아님!**
- ad_archive_id (INTEGER): 광고 아카이브 ID

- is_active (BOOLEAN): 활성 여부. 수집 시점 기준이라 과거 데이터도 TRUE일 수 있음
- end_date (FLOAT): 종료일 (숫자, 사용 비권장)

- start_date (FLOAT): 시작일 (숫자, 사용 비권장 — 사용자가 읽기 어려움)
- start_date_formatted (TIMESTAMP): 시작일 — **날짜 필터/정렬에 반드시 이 컬럼 사용!** TIMESTAMP 타입이므로 DATE()로 래핑 필요
- end_date_formatted (TIMESTAMP): 종료일 포맷됨
- ad_library_url (STRING): 광고 라이브러리 URL
- url (STRING): 광고 URL
- page_id (INTEGER): 페이지 ID
- categories (STRING): 광고 카테고리
- currency (STRING): 통화
- total_active_time (FLOAT): 총 활성 시간

### ⛔ 필수 규칙 (QA 검증 결과 반영 — 위반 시 오답 처리됨!)

1. **brand는 반드시 소문자**: `brand = 'skin1004'` ✅ | `brand LIKE '%SKIN1004%'` ❌ — 테이블에 brand 값이 전부 소문자로 저장됨. 대문자로 비교하면 0건 반환!

2. **publisher_platform은 리스트 → LIKE 필수**: facebook, instagram, messenger, audience_network 등이 쉼표로 조합된 리스트. `LOWER(publisher_platform) LIKE '%facebook%'` ✅ | `publisher_platform = 'Facebook'` ❌ (등호 비교 시 0건 반환!)
   - 정확한 개별 플랫폼 카운팅: `UNNEST(JSON_EXTRACT_ARRAY(publisher_platform)) AS platform` 사용

3. **불필요한 brand WHERE 넣지 마라**: "국가별 광고 수", "전체 현황", "유형별 분포", "활성 비율" 등 전체 집계 질문에는 `WHERE brand = 'skin1004'` 절대 추가 금지! 사용자가 명시적으로 "SKIN1004의..." 라고 말할 때만 brand 조건 추가

4. **start_date_formatted는 TIMESTAMP → DATE() 래핑 필수**:
   `WHERE DATE(start_date_formatted) >= DATE_SUB(CURRENT_DATE(), INTERVAL 7 DAY)` ✅
   `WHERE start_date_formatted >= DATE_SUB(CURRENT_DATE(), INTERVAL 7 DAY)` ❌ (타입 불일치 에러!)

5. **간단하게 생각하라**: "한국 SKIN1004 메타 광고" → `SELECT * FROM ... WHERE country_name = 'South Korea' AND brand = 'skin1004'` 충분. 불필요한 GROUP BY, CASE WHEN, 복잡한 집계 넣지 말 것!

6. **"전체 현황/요약/분석" = 전체 집계**: brand별 GROUP BY 하지 말고 total 집계로. 사용자가 "브랜드별 현황"이라고 명시할 때만 GROUP BY brand 사용

7. **이미지/비디오 구분 = snapshot 컬럼**: `JSON_VALUE(snapshot, '$.display_format') = 'VIDEO'` ✅ | ad_type으로 이미지/비디오 구분 ❌ (ad_type은 official/partnership 구분!)

8. **다른 테이블 혼동 금지**: 메타 광고 질문에 SALES_ALL_Backup, integrated_advertising_data, Platform_Data 등 다른 테이블 사용 절대 금지! 반드시 `ad_data.meta data_test`만 사용

9. **NULL/unknown 제외**: 최근 광고 리스트 출력 시 `WHERE brand IS NOT NULL AND brand != 'unknown'` 추가. ORDER BY에서 NULL은 뒤로: `NULLS LAST`

10. **country_name 값은 영어**: 'South Korea' ✅ | '한국' ❌. 변수값에 한국어 거의 없음

11. **"가장 많은 것" = LIMIT 1**: "가장 많은 유형", "1위 국가" 등 → ORDER BY DESC LIMIT 1

12. **최근 N일/주/월 = collect_date 기준**: "최근 2주 광고" → `WHERE collect_date >= DATE_SUB(CURRENT_DATE(), INTERVAL 14 DAY)` 사용. 사용자는 최근 데이터에 관심 있음

13. **페이지별 최근 광고 = ROW_NUMBER**: `ROW_NUMBER() OVER (PARTITION BY page_name ORDER BY start_date_formatted DESC) AS rn` → `WHERE rn = 1`

14. **멀티 플랫폼 분석 = UNNEST**: publisher_platform 리스트를 개별 플랫폼으로 풀기: `UNNEST(JSON_EXTRACT_ARRAY(publisher_platform)) AS platform` → GROUP BY platform

15. **답변에 null 행 출력하지 말 것**: page_name, brand 등이 null인 행은 사용자에게 무의미. 결과에서 제외

16. **질문이 애매하면 되물어볼 것**: "활성 상태 변경 내역" 같이 범위가 넓은 질문은 구체적으로 무엇이 필요한지 역질문

### 예시 쿼리

#### MT1. 국가별 메타 광고 수 (WHERE 없이 GROUP BY)
```sql
-- "국가별 메타 광고 수" → 전체 집계, WHERE brand 넣지 않음!
SELECT country_name, COUNT(*) AS ad_count
FROM `skin1004-319714.ad_data.meta data_test`
GROUP BY country_name
ORDER BY ad_count DESC
LIMIT 200
```

#### MT2. 특정 국가 + SKIN1004 광고 (간단하게!)
```sql
-- "한국 SKIN1004 메타 광고" → SELECT * + WHERE 2개 조건. 복잡하게 하지 마!
SELECT *
FROM `skin1004-319714.ad_data.meta data_test`
WHERE country_name = 'South Korea'
  AND brand = 'skin1004'
LIMIT 200
```

#### MT3. 광고 유형(official/partnership) 분포
```sql
-- ad_type = 공식/파트너십 구분. 이미지/비디오 구분 아님!
SELECT ad_type, COUNT(*) AS ad_count
FROM `skin1004-319714.ad_data.meta data_test`
GROUP BY ad_type
ORDER BY ad_count DESC
```

#### MT4. 활성/비활성 비교 (brand WHERE 없이!)
```sql
SELECT
  CASE WHEN is_active = TRUE THEN 'Active' ELSE 'Inactive' END AS ad_status,
  COUNT(*) AS ad_count
FROM `skin1004-319714.ad_data.meta data_test`
GROUP BY ad_status
```

#### MT5. 플랫폼별 광고 수 (LIKE 필수!)
```sql
-- publisher_platform은 리스트! LIKE 사용, = 등호 금지!
SELECT COUNT(*) AS facebook_ad_count
FROM `skin1004-319714.ad_data.meta data_test`
WHERE LOWER(publisher_platform) LIKE '%facebook%'
```

#### MT6. 전체 현황 리포트 (brand GROUP BY 하지 말 것!)
```sql
-- "전체 현황" → 전체 집계! brand별 분류 아님!
SELECT
  COUNT(*) AS total_ads,
  COUNTIF(is_active = TRUE) AS active_ads,
  COUNTIF(is_active = FALSE) AS inactive_ads,
  COUNT(DISTINCT brand) AS total_brands,
  COUNT(DISTINCT country_name) AS total_countries,
  COUNT(DISTINCT page_name) AS total_pages
FROM `skin1004-319714.ad_data.meta data_test`
```

#### MT7. 최근 N일 광고 (DATE() 래핑 필수!)
```sql
-- start_date_formatted는 TIMESTAMP → DATE() 래핑해야 비교 가능!
SELECT brand, page_name, ad_type, start_date_formatted, is_active, url
FROM `skin1004-319714.ad_data.meta data_test`
WHERE DATE(start_date_formatted) >= DATE_SUB(CURRENT_DATE(), INTERVAL 7 DAY)
ORDER BY start_date_formatted DESC
LIMIT 200
```

#### MT8. 비디오 유형 광고 (snapshot 컬럼 활용!)
```sql
-- 이미지/비디오 = snapshot의 display_format. ad_type 아님!
SELECT COUNT(*) AS video_meta_ads
FROM `skin1004-319714.ad_data.meta data_test`
WHERE JSON_VALUE(snapshot, '$.display_format') = 'VIDEO'
```

#### MT9. 활성 광고 비율 (전체 기준, brand 조건 없이!)
```sql
SELECT SAFE_DIVIDE(COUNTIF(is_active = TRUE), COUNT(*)) AS active_ad_ratio
FROM `skin1004-319714.ad_data.meta data_test`
```

#### MT10. 플랫폼별 정확한 카운팅 (UNNEST 사용)
```sql
-- publisher_platform 리스트를 개별로 풀어서 정확한 플랫폼별 카운팅
SELECT platform, COUNT(*) AS ad_count
FROM `skin1004-319714.ad_data.meta data_test`,
UNNEST(JSON_EXTRACT_ARRAY(publisher_platform)) AS platform
GROUP BY platform
ORDER BY ad_count DESC
```

#### MT11. 페이지별 최근 광고 (ROW_NUMBER 사용)
```sql
SELECT * FROM (
  SELECT *, ROW_NUMBER() OVER (PARTITION BY page_name ORDER BY start_date_formatted DESC) AS rn
  FROM `skin1004-319714.ad_data.meta data_test`
) WHERE rn = 1
```

#### MT12. 2025년 특정 월 광고 (start_date_formatted 사용!)
```sql
-- start_date(FLOAT) 아님! start_date_formatted(TIMESTAMP) 사용!
SELECT *
FROM `skin1004-319714.ad_data.meta data_test`
WHERE DATE(start_date_formatted) BETWEEN '2025-02-01' AND '2025-02-28'
```

#### MT13. 최근 광고 N건 (null/unknown 제외!)
```sql
SELECT collect_date, country_name, brand, page_name, ad_type, is_active, url
FROM `skin1004-319714.ad_data.meta data_test`
WHERE brand IS NOT NULL AND brand != 'unknown'
ORDER BY collect_date DESC
LIMIT 20
```

#### MT14. 멀티 플랫폼 게시 광고
```sql
SELECT *
FROM `skin1004-319714.ad_data.meta data_test`
WHERE ARRAY_LENGTH(SPLIT(publisher_platform, ',')) > 1
LIMIT 200
```
