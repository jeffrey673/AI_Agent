## 규칙

1. **SELECT 문만 사용** — INSERT, UPDATE, DELETE, DROP 절대 금지
2. **BigQuery SQL 문법 준수**:
   - 날짜: `EXTRACT(YEAR FROM Date)`, `EXTRACT(MONTH FROM Date)`, `DATE(Date)`, `FORMAT_DATETIME('%Y-%m', Date)`
   - 안전 나눗셈: `SAFE_DIVIDE(a, b)`
   - 문자열: `LOWER()`, `UPPER()`, `CONTAINS_SUBSTR()`
3. **테이블명은 반드시 백틱(`)으로 감싸기**: `skin1004-319714.Sales_Integration.SALES_ALL_Backup`
4. **결과는 1000행 이내로 LIMIT 추가**
5. ⚠️ **Country 값은 한국어** — 영어 아님! 변환 불필요
6. ⚠️ **Date는 DATETIME** — DATE 함수로 감싸거나 EXTRACT 사용
7. ⚠️ **매출 조회 시 Sales1_R 사용** (원화 환산 매출). Sales2_R은 Sales1_R이 없을 때만 보조
8. ⚠️ **수량 조회 시 Total_Qty 사용** (Quantity 컬럼 사용 금지)
9. ⚠️ **제품명 조회 시 `\`SET\`` 컬럼 사용** (Product_Name 사용 금지). SET은 SQL 예약어이므로 반드시 백틱(`)으로 감싸기!
10. ⚠️ **대륙 조회 시 Continent1 사용** (Continent2는 상세 지역 필요 시만)
11. ⚠️ **팀 조회 시 Team_NEW 사용** (Team 사용 금지)
12. **플랫폼 필터링은 Mall_Classification으로** — LIKE 사용
13. ⚠️ **1900년대 데이터 제외** — 모든 쿼리에 `Date >= '2000-01-01'` 조건을 반드시 추가하세요. 1900년대 데이터는 목표/플래너 데이터이므로 실제 매출 분석에서 제외해야 합니다.
14. ⚠️ **시각화/차트/그래프 요청 시 반드시 GROUP BY로 집계**:
   - 사용자가 "시각화", "차트", "그래프", "추이", "비교", "보여줘" 등을 요청하면, 반드시 SUM/COUNT 등의 집계 함수와 GROUP BY를 사용해서 요약된 결과를 반환하세요.
   - 결과가 차트로 표현 가능하도록 행 수를 적절히 제한하세요 (보통 5~30행).
   - 월별 추이: `FORMAT_DATETIME('%Y-%m', Date) AS month` + `GROUP BY month ORDER BY month`
   - 국가별 비교: `Country` + `GROUP BY Country ORDER BY total_revenue DESC`
   - 절대 집계 없이 raw 행 데이터를 반환하지 마세요!
   - ⚠️ Date 필터 시 반드시 `Date >= '2000-01-01'` 포함!

```sql
  AND EXTRACT(YEAR FROM Date) = 2024
  AND EXTRACT(MONTH FROM Date) = 1

```

### 2. 국가+플랫폼 기간 매출
```sql
-- "2024년 상반기 인도네시아 쇼피 매출"
SELECT
  FORMAT_DATETIME('%Y-%m', Date) AS month,
  SUM(Sales1_R) AS monthly_revenue,
  SUM(Total_Qty) AS total_quantity
FROM `skin1004-319714.Sales_Integration.SALES_ALL_Backup`
WHERE Country = '인도네시아'
  AND Mall_Classification LIKE '%Shopee%'
```
