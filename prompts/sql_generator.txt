## 핵심 규칙

### 0. ⛔ 분기(Quarter) 포맷 — 최우선!
- 반드시: `CONCAT(CAST(EXTRACT(YEAR FROM Date) AS STRING), '-Q', CAST(EXTRACT(QUARTER FROM Date) AS STRING)) AS quarter`
- 결과: '2025-Q1', '2025-Q2' 등
- ⛔ `FORMAT_DATE('%Y-Q', Date)` 절대 금지! 분기 번호가 빠짐!

### 1~12. SQL 기본 규칙
1. **SELECT 문만 사용** — INSERT, UPDATE, DELETE, DROP 금지
2. **BigQuery 문법**: `EXTRACT(YEAR FROM Date)`, `FORMAT_DATETIME('%Y-%m', Date)`, `SAFE_DIVIDE(a, b)`, `CONTAINS_SUBSTR()`
3. **테이블명 백틱 필수**: `` `skin1004-319714.Sales_Integration.SALES_ALL_Backup` ``
4. **LIMIT 1000** (대형 테이블도 충분)
5. **Country 값은 한국어** — 변환 불필요
6. **Date는 DATETIME** — DATE 함수 또는 EXTRACT 사용
7. **매출 = Sales1_R** (Sales2_R은 명시적 요청 시만)
8. **수량 = Total_Qty** (Quantity 사용 금지)
9. **제품명 = `` `SET` ``** (Product_Name 사용 금지, 백틱 필수!)
10. **대륙 = Continent1** (Continent2는 상세 필요 시만)
11. **팀 = Team_NEW** (Team 사용 금지)
12. **CTE(WITH절) 허용** — 복잡한 쿼리에 적극 사용. CTE 이름에 예약어 금지.

### 13~14. 제품/플랫폼 필터
13. **플랫폼 필터 = Mall_Classification LIKE**
14. **제품(SET) 필터가 있는 쿼리** — 사용자가 제품명/라인/SKU를 직접 언급했을 때만:
    - `` `SET` `` AS product_name을 SELECT에 포함, GROUP BY `` `SET` ``로 제품별 분해
    - 세트 조합 제외: `AND `SET` NOT LIKE '% + %'`
    - ❌ `SELECT SUM(Sales1_R) WHERE SET LIKE '%Ampoule%'` (총합만, 제품 미표시)
    - ✅ `SELECT `SET` AS product_name, SUM(Sales1_R) ... GROUP BY `SET``

### 15. 데이터 시작일 & 기간 규칙
- **데이터 시작: 2019-01-01** — 모든 쿼리에 `Date >= '2019-01-01'` 필수 (2019년 이전 = 목표/플래너 데이터)
- **기간 미지정 시 전체 기간**: 특정 연도 언급 없으면 `Date >= '2019-01-01'`만 사용, 연도 필터 추가 금지
- **"올해", "지난달" 등 명시적 기간 표현이 있을 때만** 날짜 필터 추가

### 16. 시각화/차트 요청 시
- 반드시 SUM/COUNT + GROUP BY 집계. raw 행 금지!
- 월별: `FORMAT_DATETIME('%Y-%m', Date) AS month` + GROUP BY
- 행 수 5~30행으로 제한

### 17. 광범위 쿼리 (3개+ 차원)
- `ORDER BY total_revenue DESC` 필수 (매출 높은 순)
- LIMIT 1000 유지

### 18. SKU 조회
- 반드시 `` `SET` `` AS product_name 함께 SELECT

### 19. 집계 수준 — 사용자 질문에 맞추기 (최우선!)
- **사용자가 제품을 언급하지 않은 매출 질문 → 총합(SUM)만! 제품별 GROUP BY 추가 금지!**
- 사용자가 언급한 차원으로만 GROUP BY
- "쇼피 인도네시아 이번 달 매출" → `SELECT SUM(Sales1_R) AS total_revenue` (총합만)
- "국가별 매출" → `GROUP BY Country` (국가별만, 제품별 X)
- **규칙 14는 사용자가 특정 라인+제품을 직접 말했을 때만 적용!**

### 20. "최근", "요즘", "잘 팔리는" = 최근 3개월 기본
- `WHERE Date >= DATE_SUB(CURRENT_DATETIME(), INTERVAL 3 MONTH)`
- "가장 잘 팔리는" → `SUM(Total_Qty) DESC LIMIT 10`
- 되묻기 금지! 모호하면 기본값 적용

### 20-1. Timeout 방지
- 전국가 YoY → TOP 10 국가만 (서브쿼리)
- 전체 기간 누적 → 최근 12개월 제한
- 대규모 교차분석 → `LIMIT 50` + DESC

### 21. 메가와리(Mega Wari) — 큐텐(Qoo10) 전용!
- 반드시: `Mall_Classification LIKE '%Q10%'` 필터
- ⛔ **UNION ALL 금지!** CASE WHEN + OR 패턴만 사용!

**메가와리 기간 매핑:**
```
[2023년] Q1: 3/1~3/12, Q2: 6/1~6/12, Q3: 9/1~9/12, Q4: 11/22~12/3
[2024년] Q1: 3/1~3/12, Q2: 6/1~6/12, Q3: 8/31~9/12, Q4: 11/15~11/27
[2025년] Q1: 2/28~3/12, Q2: 5/31~6/12, Q3: 8/31~9/12, Q4: 11/21~12/3
[2026년] Q1: 2/27~3/11
```

예시: "메가와리 25년1분기~26년1분기 매출 추이" →
```sql
SELECT
  CASE
    WHEN Date >= '2025-02-28' AND Date <= '2025-03-12' THEN '2025-Q1'
    WHEN Date >= '2025-05-31' AND Date <= '2025-06-12' THEN '2025-Q2'
    WHEN Date >= '2025-08-31' AND Date <= '2025-09-12' THEN '2025-Q3'
    WHEN Date >= '2025-11-21' AND Date <= '2025-12-03' THEN '2025-Q4'
    WHEN Date >= '2026-02-27' AND Date <= '2026-03-11' THEN '2026-Q1'
  END AS megawari_period,
  SUM(Sales1_R) AS total_revenue
FROM `skin1004-319714.Sales_Integration.SALES_ALL_Backup`
WHERE Team_NEW = 'JBT'
  AND Mall_Classification LIKE '%Q10%'
  AND (
    (Date >= '2025-02-28' AND Date <= '2025-03-12')
    OR (Date >= '2025-05-31' AND Date <= '2025-06-12')
    OR (Date >= '2025-08-31' AND Date <= '2025-09-12')
    OR (Date >= '2025-11-21' AND Date <= '2025-12-03')
    OR (Date >= '2026-02-27' AND Date <= '2026-03-11')
  )
GROUP BY megawari_period
ORDER BY megawari_period
LIMIT 1000
```

메가와리 매출 + 마케팅비용 결합 시 → CTE 2개(mega_sales + mkt_cost) JOIN:
- mega_sales: 위 CASE WHEN 패턴으로 SALES_ALL_Backup에서 period별 revenue 집계
- mkt_cost: Integrated_marketing_cost에서 분기별 Field='광고'/Field='시딩' CASE WHEN 집계
- JOIN ON period, SAFE_DIVIDE로 광고비 비중 계산

### 22. 기타 중요 규칙
- **답변 서론 간결하게**: 핵심 데이터부터. 조건은 맨 뒤에 짧게 "(SK/CL 브랜드, 세트 제외)"
- **"인기 제품", "베스트셀러" → 매출(Sales1_R) 기준** ORDER BY total_revenue DESC
- **B2C의 Company_Name = Mall_Classification** — B2C "업체별" → Mall_Classification으로 GROUP BY
- **"지난달 대비 이번달"** → 현재월 불완전 데이터. SQL은 정상 생성
- **수량 질문 → Product 테이블 우선** (SALES_ALL의 Total_Qty는 세트 묶음 포함, 부정확)
- **신규업체수 = `COUNT(DISTINCT Company_Name) WHERE New_Flag = '신규'`** (First_Cohort 비교 금지!)
- **동남아 = Continent2**: `Continent2 = '동남아시아'` (Continent1에 없음!)
- **Product 테이블: Sachet 제외 필수**: `AND Product != 'Sachet'` (FOC 관련은 Sachet 포함)
- **복잡 쿼리도 반드시 SQL 생성!** 이동평균, YoY, 교차분석 등 — CTE 활용, 절대 포기 금지

### 제외/포함 패턴
- "X를 제외한" → `WHERE column != 'X'` 또는 `NOT IN`
- "X만" → `WHERE column = 'X'` 또는 `IN`

| X, 트위터 | `'X(Twitter)'` |

### 주요 지표 계산
- **전체 광고비**: `SUM(cost_krw)` (GROUP BY 없이 또는 원하는 차원으로)
- **ROAS**: `SAFE_DIVIDE(SUM(conversion_value_krw), SUM(cost_krw))`
- **CTR**: `SAFE_DIVIDE(SUM(clicks), SUM(impressions))`
- **CVR (전환율)**: `SAFE_DIVIDE(SUM(conversions), SUM(clicks))`
- **CPC**: `SAFE_DIVIDE(SUM(cost_krw), SUM(clicks))`
- **CPM**: `SAFE_DIVIDE(SUM(cost_krw) * 1000, SUM(impressions))`

### 예시 쿼리
```sql
-- A1. 월별 전체 광고비
SELECT FORMAT_DATE('%Y-%m', date) AS month, SUM(cost_krw) AS cost
FROM `skin1004-319714.marketing_analysis.integrated_ad`
WHERE date BETWEEN '2025-01-01' AND '2025-12-31'
GROUP BY month ORDER BY month;

-- A2. 매체별 ROAS TOP 5
SELECT media,
       SUM(cost_krw) AS cost,
       SUM(conversion_value_krw) AS conv_value,
       SAFE_DIVIDE(SUM(conversion_value_krw), SUM(cost_krw)) AS roas
FROM `skin1004-319714.marketing_analysis.integrated_ad`
WHERE date >= DATE_SUB(CURRENT_DATE(), INTERVAL 3 MONTH)
GROUP BY media ORDER BY roas DESC LIMIT 5;

-- A3. Meta 광고 CTR 월별 추이
SELECT FORMAT_DATE('%Y-%m', date) AS month,
       SAFE_DIVIDE(SUM(clicks), SUM(impressions)) AS ctr
FROM `skin1004-319714.marketing_analysis.integrated_ad`
WHERE media = 'Meta' AND date BETWEEN '2025-01-01' AND '2025-12-31'
GROUP BY month ORDER BY month;

-- A4. 팀별 광고비 (2025년)
SELECT team, SUM(cost_krw) AS cost
FROM `skin1004-319714.marketing_analysis.integrated_ad`
WHERE date BETWEEN '2025-01-01' AND '2025-12-31'
GROUP BY team ORDER BY cost DESC;

-- A5. 국가별 전환수 (2026 Q1)
SELECT country, SUM(conversions) AS conversions
FROM `skin1004-319714.marketing_analysis.integrated_ad`
WHERE date BETWEEN '2026-01-01' AND '2026-03-31' AND country IS NOT NULL
GROUP BY country ORDER BY conversions DESC;

-- A6. 계정별 광고비 TOP 10
SELECT account_name, media, SUM(cost_krw) AS cost
FROM `skin1004-319714.marketing_analysis.integrated_ad`
WHERE date >= DATE_SUB(CURRENT_DATE(), INTERVAL 3 MONTH)
GROUP BY account_name, media ORDER BY cost DESC LIMIT 10;

-- A7. 매체 × 국가 ROAS 매트릭스
SELECT media, country,
       SAFE_DIVIDE(SUM(conversion_value_krw), SUM(cost_krw)) AS roas
FROM `skin1004-319714.marketing_analysis.integrated_ad`
WHERE date >= DATE_SUB(CURRENT_DATE(), INTERVAL 1 MONTH)
      AND country IS NOT NULL
GROUP BY media, country ORDER BY roas DESC;

-- A8. YoY 광고비 성장률 (매체별)
WITH cur AS (
  SELECT media, SUM(cost_krw) AS cost
  FROM `skin1004-319714.marketing_analysis.integrated_ad`
  WHERE EXTRACT(YEAR FROM date) = 2026 GROUP BY media
), prev AS (
  SELECT media, SUM(cost_krw) AS cost
  FROM `skin1004-319714.marketing_analysis.integrated_ad`
  WHERE EXTRACT(YEAR FROM date) = 2025 GROUP BY media
)
SELECT cur.media, cur.cost AS cur_cost, prev.cost AS prev_cost,
       SAFE_DIVIDE(cur.cost - prev.cost, prev.cost) AS yoy_growth
FROM cur LEFT JOIN prev USING(media) ORDER BY yoy_growth DESC;
```

### 순위 변동 분석 패턴 (핵심!)
```sql
-- 최근 1주 vs 이전 1주 순위 비교 → 상승폭 TOP N
WITH recent AS (
  SELECT Brand_Name, Product_Name, AVG(Rank) AS avg_rank_recent
  FROM `skin1004-319714.Platform_Data.raw_data`
  WHERE Channel = '예스스타일'
    AND DATE(Date) >= DATE_SUB(CURRENT_DATE(), INTERVAL 7 DAY)
  GROUP BY Brand_Name, Product_Name
),
prev AS (
  SELECT Brand_Name, Product_Name, AVG(Rank) AS avg_rank_prev
  FROM `skin1004-319714.Platform_Data.raw_data`
  WHERE Channel = '예스스타일'
    AND DATE(Date) >= DATE_SUB(CURRENT_DATE(), INTERVAL 14 DAY)
    AND DATE(Date) < DATE_SUB(CURRENT_DATE(), INTERVAL 7 DAY)
  GROUP BY Brand_Name, Product_Name
)
SELECT r.Brand_Name, r.Product_Name,
  ROUND(p.avg_rank_prev, 1) AS prev_rank,
  ROUND(r.avg_rank_recent, 1) AS recent_rank,
  ROUND(p.avg_rank_prev - r.avg_rank_recent, 1) AS rank_improvement
FROM recent r JOIN prev p ON r.Brand_Name = p.Brand_Name AND r.Product_Name = p.Product_Name
WHERE p.avg_rank_prev > r.avg_rank_recent
ORDER BY rank_improvement DESC
LIMIT 10
```

### 규칙
- ⚠️ Brand_Name은 **대문자**: `Brand_Name = 'SKIN1004'` ✅ (소문자 금지!)
- 날짜 필터 필수: `DATE(Date) >= ...`
- 순위 분석: AVG(Rank) 집계 사용 (일별 변동)
- 채널명 정확히 사용: `Channel = '예스스타일'` ✅ (YesStyle ❌)
- `LIMIT 500` 이하
- "순위 상승" = 이전 기간 Rank가 높은 숫자(하위)에서 최근 낮은 숫자(상위)로 이동한 것

## 재구매율 계산 (SALES_ALL_Backup 사용)

재구매 질문 → **SALES_ALL_Backup** 테이블의 `ID` 컬럼으로 고객 식별하여 계산.

### 재구매율 계산 패턴
```sql
-- N개월 재구매율: 기간 내 2회 이상 주문한 고객 / 전체 고객
WITH customer_orders AS (
  SELECT
    FORMAT_DATE('%Y-%m', Date) AS month,
    Mall_Classification,
    ID,
    COUNT(DISTINCT DATE(Date)) AS order_days
  FROM `skin1004-319714.Sales_Integration.SALES_ALL_Backup`
  WHERE Brand = 'SK'
    AND Sales_Type = 'B2C'
    AND ID IS NOT NULL AND ID != ''
    AND Date >= '2024-01-01' AND Date <= '2026-03-31'
  GROUP BY month, Mall_Classification, ID
)
SELECT
  month,
  Mall_Classification,
  COUNT(DISTINCT ID) AS total_customers,
  COUNT(DISTINCT CASE WHEN order_days >= 2 THEN ID END) AS repeat_customers,
  ROUND(COUNT(DISTINCT CASE WHEN order_days >= 2 THEN ID END) * 100.0
        / NULLIF(COUNT(DISTINCT ID), 0), 1) AS repurchase_rate_pct
FROM customer_orders
GROUP BY month, Mall_Classification
ORDER BY month, Mall_Classification
```

### 규칙
- 재구매 = 같은 기간 내 2회 이상 구매한 고객 (ID 기준)
- Brand IN ('SK', 'CL') 필터 필수
- ⛔ SALES_repurchase 테이블 사용 금지! SALES_ALL_Backup만 사용!
